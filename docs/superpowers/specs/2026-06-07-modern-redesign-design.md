# Современный редизайн сайта ТБДТ — дизайн-спецификация

**Дата:** 2026-06-07
**Проект:** Сайт продажи билетов Тюменского БДТ (публичная страница + кабинет администратора)
**Цель:** Пересобрать дизайн всего проекта в современном ключе — со сдержанными плавными анимациями, строгим гротеском (без засечек), сохранив театральный характер. Светлая тема. Всё логично и стильно.

## Контекст и ограничения

- Статический проект: `index.html`, `admin.html`, `styles.css`, `admin.css`, `app.js`, `admin.js`, `assets/` (`cover.webp`, `logo.png`).
- Логика в `app.js` и `admin.js` завязана на конкретные `id` и классы DOM. **Все JS-хуки (id и классы, на которые ссылается JS) сохраняются** — иначе ломается функциональность. Редизайн идёт прежде всего через CSS + тонкий слой анимаций; правки HTML допустимы только с сохранением этих хуков.
- Текущий стиль («винтажный театральный плакат»: бумага, зерно, виньетка, засечный Cormorant, навигационные курсивы) **заменяется** на чистую современную светлую тему.
- Поддержка `prefers-reduced-motion` обязательна — при включённом режиме анимации отключаются/сводятся к мгновенным переходам.

## 1. Дизайн-язык (общий для обеих страниц)

### Палитра (светлая тема)
- Фон: `#FAFAF8` (тёплый почти-белый)
- Поверхности/карточки: `#FFFFFF`
- Текст основной: `#16161A`; вторичный: `#6B6B73`
- Акцент (навигация, primary-кнопки): navy `#1E2042`, hover `#2C2E5C`
- Золото `#E6B81C` — точечно: тонкие линии-акценты, цифры-нумерация секций, активные состояния, «золотые» кнопки в админке
- Границы: `rgba(20,20,30,.08)`
- Тени: мягкие, многослойные; **grain и vignette удаляются**

### Типографика
- Заголовки: **Unbounded** (крупно, плотный трекинг)
- Интерфейс/текст: **Rubik** (обновлено: изначально планировался Manrope)
- Удаляются: засечный Cormorant Garamond и винтажные курсивы. Театральность достигается ритмом, акцентами и нумерацией секций, а не засечками.
- Шрифты подключаются через Google Fonts (заменить текущий `Rubik`/`Unbounded` link на `Manrope`/`Unbounded`).

### Геометрия и тон
- Радиусы: 12–20px (чуть меньше текущих)
- Шкала отступов: 4 / 8 / 12 / 16 / 24 / 32 / 48
- Больше воздуха, чёткая сетка, последовательные отступы

### Анимации (сдержанные, плавные)
- Scroll-reveal: fade + slide вверх, со stagger по элементам (IntersectionObserver)
- Hover: мягкий подъём кнопок и карточек
- Выбор мест в зале: плавное pop-изменение состояния
- Модалки/вкладки: аккуратные переходы появления (fade + лёгкий подъём)
- Реализация — `transition` + IntersectionObserver; уважение к `prefers-reduced-motion`

## 2. Публичная страница (index.html)

- **Хедер:** прозрачный, `backdrop-blur`, тонкая нижняя граница; уплотняется при скролле. Лого слева, навигация, кнопка корзины с золотым бейджем-счётчиком. Анимированное подчёркивание ссылок.
- **Hero:** full-bleed `cover.webp` + чистый светлый градиент-скрим слева (без виньетки). Слева: кикер, крупный заголовок Unbounded в две строки, подзаголовок, две кнопки (navy primary + ghost), строка фактов (дата / время / длительность / сцена) с тонкими разделителями. На загрузке — stagger-reveal строк заголовка и фактов + плавный масштаб фото. Без параллакса (ради сдержанности).
- **Лента дат:** горизонтальные карточки-«таблетки» (число / месяц / день недели); активная — navy-заливка + золотая нижняя метка. Под ней session-bar: выбранный сеанс + времена-чипы.
- **Зал (ключевой блок):** двухколоночно — слева схема, справа sticky-корзина.
  - Сцена: тонкая дуга с мягким свечением (без неонового glow).
  - Места — кружки: свободно (контур), выбрано (navy-заливка + лёгкое pop-увеличение), занято (приглушённый серый). Hover — золотой контур + floating tooltip.
  - Корзина: белая карточка с тенью; список мест со slide-in при добавлении; итоги; кнопка «Оформить»; легенда чипами.
- **О театре:** крупная цитата + три колонки (Адрес / Касса / Возврат) на тонких линиях.
- **Футер:** минималистичный, одна строка, тонкая верхняя граница.
- **Модалка оформления:** плавное появление (fade + подъём панели). Две колонки — форма слева, сводка справа. Поля с navy focus-кольцом. Экран успеха с «билетом» (перфорация-tear — тонкая пунктирная линия).

## 3. Кабинет администратора (admin.html)

- **Сайдбар:** белая панель, тонкая правая граница. Лого, профиль (золотой круг-аватар с инициалами), вертикальная навигация с иконками; активный пункт — navy-заливка + золотая метка слева, плавный переход. Внизу — «Вернуться на сайт».
- **Топ-бар:** хлебные крошки + крупный заголовок Unbounded (меняется по вкладке). Справа сегмент-контрол периода (Неделя / Месяц / Сезон) с плавным «пилюлей»-индикатором под активным.
- **Баннер финстатуса:** широкая карточка с цвет-кодировкой состояния (рост — мягкий зелёный, спад — приглушённый красный, нейтрально — navy). Стрелка-маркер, текст, крупная цифра справа. Появление fade + slide.
- **Обзор:** ряд KPI-карточек (число крупно Unbounded, подпись Manrope, тонкая динамика), reveal со stagger. График выручки — столбики с плавным «вырастанием». Гейдж заполняемости — кольцевой индикатор с анимацией заполнения + список зон.
- **Цены:** чистая таблица на тонких линиях (без zebra), цветные точки зон, инлайн-редактирование цены, золотая кнопка «+ Добавить тариф». Подвал: прогноз / план / статус; при пересчёте — лёгкая подсветка изменившейся цифры.
- **Залы:** палитра тарифов-чипов сверху; редактор-сетка мест с полосой «СЦЕНА»; назначение зон кликом/перетаскиванием; сводка справа. Состояния мест — единый язык с публичной схемой.
- **Аналитика:** панели с графиками (средний чек), горизонтальные бары по зонам и каналам с анимацией ширины при reveal.
- **Бухгалтерия:** сводные карточки + таблица операций со статус-чипами (оплачено / ожидание / расход).
- **Тосты:** общий компонент для обеих страниц — слайд снизу, авто-скрытие.

## Подход к реализации

1. Переписать `styles.css` (публичная) и `admin.css` (кабинет) под новый дизайн-язык; вынести общие токены (палитра, шрифты, радиусы, тени, easing) в `:root`, единые на обеих страницах.
2. Обновить подключение шрифтов в `index.html` и `admin.html` (Manrope + Unbounded); удалить разметку grain/vignette.
3. Добавить тонкий слой анимаций: IntersectionObserver для scroll-reveal + CSS-переходы. Минимальные правки JS только для reveal-хуков и при необходимости синхронизации классов; существующие функциональные `id`/классы не трогать.
4. Проверить обе страницы в браузере: вёрстка, состояния (выбор мест, модалка, переключение вкладок), `prefers-reduced-motion`, адаптив.

## Критерии готовности

- Обе страницы выглядят современно, в единой светлой палитре navy + gold, со шрифтами Rubik (UI) + Unbounded (заголовки), без засечек, без grain/vignette.
- Анимации сдержанные, плавные, не ломают использование; отключаются при `prefers-reduced-motion`.
- Вся существующая функциональность (выбор мест, корзина, модалка оформления, вкладки админки, редактор зала, графики) работает как прежде.
- Адаптив сохранён/улучшен.
